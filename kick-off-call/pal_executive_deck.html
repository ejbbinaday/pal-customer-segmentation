Under The Hood

How Machine Learning Labels Every Passenger

Stages 3–6  ·  Fully explainable  ·  Every label traceable to booking signals  ·  No black box

①
Proxy Label Assignment (Priority Waterfall)
9 Business Rules Tag 76% Instantly
Human-defined rules run first — before any Machine Learning. J cabin → Corporate  ·  ME route → OFW/Migrant  ·  ≤3-day booking → Last-Minute  ·  Group via TTA → Pilgrimage
22,907 passengers labelled · 76.4%
②
Clustering Algorithm Evaluation
7 Algorithms Tested — HDBSCAN Selected
Remaining 24% grouped by booking similarity. HDBSCAN follows natural density patterns, does not force equal-sized groups, and flags borderline passengers rather than guessing.
78 natural clusters · 7.1% borderline flagged
③
Cluster → Segment Mapping & Noise Assignment
78 Clusters Mapped to 10 Named Segments
Each cluster matched to its nearest named segment using the same booking signals. Borderline passengers auto-assigned. Every label is fully traceable to booking signals.
